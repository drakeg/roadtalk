# Internet CB / RoadTalk Development Plan

## Development Methodology

Follow Agile with Scrum-style sprints and a Kanban board. Each sprint must be completed and accepted before the next begins. The current sprint is the only sprint under active development. Every sprint ends with working software, automated tests, updated documentation, and a deployable repository.

## Definition of Done

- Requirements implemented.
- Acceptance tests pass.
- Unit/integration tests updated.
- Documentation updated.
- Kanban board reflects completion.
- Demo completed before next sprint.

## Agile Governance

- Maintain a Kanban board with Backlog, Ready, In Progress, Review, Testing, Done.
- Every story has acceptance criteria.
- No scope creep inside an active sprint.
- Retrospective after every sprint.
- Keep the application deployable at all times.

## Initial MVP Goal

Complete Sprints 0–5 to produce a usable Internet CB MVP for real-world RV campground testing.
